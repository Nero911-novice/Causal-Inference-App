Streamlit-приложение для оценки инкрементального эффекта маркетингового воздействия через аналитическую триангуляцию.

Приложение принимает временной ряд по целевой и контрольной группам, строит несколько контрфактуальных baseline-сценариев, сравнивает их с фактическим результатом в тестовом периоде и дополняет вывод проверками устойчивости.

## Overview

Приложение предназначено для прикладной оценки эффекта кампаний в ситуациях, где нет полноценного экспериментального дизайна, а исторический ряд ограничен по длине.

Основная логика:

1. загрузить исторический ряд по целевой и контрольной группам;
2. перевести данные в месячную динамику;
3. построить три независимые оценки baseline;
4. сравнить baseline с фактом в тестовом периоде;
5. проверить устойчивость результата к изменению исторического окна и состава данных.

- Excel-экспорт результатов.

## Methodology

### Базовая единица анализа

Приложение работает не с абсолютными значениями напрямую, а с месячной динамикой:

```text
MoM = (V_t - V_t-1) / V_t-1
```

где:

- `V_t` — значение показателя в текущем месяце;
- `V_t-1` — значение показателя в предыдущем месяце.

### Периоды

Используются два окна:

- `Historical Period` — окно для оценки исторической закономерности;
- `Test Period` — окно воздействия, для которого оценивается эффект.

### Модели baseline

#### 1. Регрессия чувствительности к рынку

Используется связь:

```text
Target_MoM = beta * Control_MoM
```

На тестовом периоде baseline строится через фактическую динамику контрольной группы и коэффициент `beta`.

#### 2. CV Leverage

Используется относительная волатильность рядов:

```text
CV = std / mean
L = CV_control / CV_target
```

Прогнозная динамика корректирует рыночное движение через коэффициент `L`.

#### 3. Сезонная модель YoY

Для каждого тестового месяца берется средний исторический `Target_MoM` для того же календарного месяца.

### Расчет эффекта

Для каждой модели считается:

```text
Impact_pp = Fact_MoM - Forecast_MoM
```

Затем baseline переводится в абсолютную контрфактуальную траекторию, после чего рассчитывается экономический эффект в абсолютных единицах.

## Input data

Ожидаются три колонки:

| Column | Description |
|---|---|
| `Date` | дата наблюдения |
| `Target_Group` | абсолютные значения целевой группы |
| `Control_Group` | абсолютные значения контрольной группы или рыночного ориентира |

Поддерживаемые форматы файлов:

- `CSV`
- `XLSX`
- `XLS`

### Что происходит после загрузки

1. даты приводятся к месячному уровню;
2. повторяющиеся строки внутри одного месяца агрегируются;
3. значения очищаются от нечисловых и пустых записей;
4. рассчитываются `Target_MoM` и `Control_MoM`.

## Robustness checks

Приложение включает четыре независимых блока проверки устойчивости.

### Placebo analysis

Создаются псевдо-тестовые окна внутри `Historical Period` той же длины, что и реальный `Test Period`.

Задача:
оценить, насколько фактический эффект выделяется на фоне ложных окон.

### Leave-one-year-out

Из historical периода поочередно исключается один календарный год.

Задача:
проверить, зависит ли вывод от одного года.

### Leave-one-month-out

Из historical периода поочередно исключается один месяц.

Задача:
проверить, зависит ли вывод от отдельной точки ряда.

### Historical boundary sensitivity

Начало и конец `Historical Period` слегка сдвигаются вокруг исходного окна.

Задача:
оценить чувствительность результата к выбору границ обучения.

## Interpretation

Интерпретация в приложении строится в двух слоях:

### 1. Quality notes

Пояснения по качеству самих моделей:

- объясняющая сила рыночной модели;
- относительная волатильность целевой группы;
- достаточность сезонной поддержки.

### 2. Robustness interpretation

Текстовая маркировка по stress-check блокам:

- сила сигнала эффекта в placebo-анализе;
- устойчивость эффекта к исключению лет;
- устойчивость эффекта к исключению месяцев;
- устойчивость к сдвигу границ `Historical Period`.

Эти выводы представляют собой прикладную аналитическую маркировку.  
Они не заменяют экспериментальную идентификацию причинности и не являются формальным статистическим доказательством.

## Limitations

При интерпретации результата важно учитывать следующие ограничения:

- контрольная группа должна быть содержательно связана с целевой;
- короткий `Historical Period` снижает устойчивость оценки;
- структурные изменения бизнеса могут искажать baseline;
- сезонная модель чувствительна к длине и полноте исторического ряда;
- эффект следует трактовать как оценку через набор контрфактуальных сценариев, а не как окончательное доказательство причинности.
