# Causal Inference App

Streamlit-приложение для оценки инкрементального эффекта маркетингового воздействия через аналитическую триангуляцию.

Приложение принимает временной ряд по целевой и контрольной группам, строит три независимых baseline-сценария, оценивает эффект в тестовом периоде и дополняет результат проверками устойчивости.

## Quick start

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Запуск приложения

```bash
streamlit run causal_inference_app.py
```

### 3. Входные данные

Файл должен содержать три колонки:

- `Date`
- `Target_Group`
- `Control_Group`

Поддерживаются форматы `CSV`, `XLSX`, `XLS`.

---

## Что делает приложение

Приложение:

- приводит даты к месячному уровню;
- агрегирует повторяющиеся строки внутри месяца;
- рассчитывает месячную динамику `MoM`;
- строит три baseline-модели;
- считает чистый эффект в п.п. и в абсолютных значениях;
- показывает таблицы, график и проверки устойчивости;
- формирует автоматическую интерпретацию результата.

---

## Логика расчета

После загрузки данных приложение рассчитывает месячную динамику:

```text
MoM = (V_t - V_t-1) / V_t-1
```

Далее анализ разбивается на два периода:

- `Historical Period` — обучение и извлечение исторической закономерности;
- `Test Period` — период воздействия, для которого оценивается эффект.

Для каждого метода строится контрфактуальный baseline: ожидаемое поведение `Target_Group`, если бы воздействия не было.

---

## Методы

### 1. Регрессия чувствительности к рынку

Используется зависимость:

```text
Target_MoM = beta * Control_MoM
```

Прогноз в тестовом периоде строится на основе фактической рыночной динамики и коэффициента `beta`.

### 2. CV Leverage

Используется относительная волатильность целевой и контрольной групп:

```text
CV = std / mean
L = CV_control / CV_target
```

Прогнозная динамика корректирует рыночное движение через коэффициент `L`.

### 3. Сезонная модель YoY

Для каждого тестового месяца используется средний исторический `Target_MoM` для того же календарного месяца.

---

## Расчет эффекта

Для каждой модели рассчитывается:

```text
Impact_pp = Fact_MoM - Forecast_MoM
```

Затем baseline переводится в абсолютную контрфактуальную траекторию, после чего считается экономический эффект в абсолютных значениях.

---

## Проверки устойчивости

В приложении реализованы:

- `Placebo-анализ`
- `Leave-one-year-out`
- `Leave-one-month-out`
- `Границы Historical`

Эти проверки позволяют оценить:

- выделяется ли фактический эффект на фоне ложных окон;
- зависит ли вывод от одного года или одного месяца;
- чувствителен ли результат к границам historical периода.

## Ограничения

Приложение предназначено для прикладной аналитической оценки и не заменяет экспериментальный причинный дизайн.

На результат влияют:

- качество контрольной группы;
- длина historical периода;
- наличие структурных изменений бизнеса;
- устойчивость baseline к выбору окна и состава истории.

Результат следует трактовать как оценку эффекта через набор контрфактуальных сценариев.

---

## Где применять

Инструмент подходит для:

- внутренней маркетинговой аналитики;
- оценки инкрементального эффекта кампаний;
- сравнения нескольких baseline-сценариев;
- работы с небольшими историческими рядами, где важны проверки устойчивости.
